## Project Structure

### Core Files
- `main.py`: Complete perceptron implementation with training, prediction, visualization, and CLI
  - Contains all core logic: step activation, predict function, train function, test function
  - Includes LOGIC_GATES dictionary with truth tables for AND, OR, NAND, NOR, XOR
  - CLI with argparse for configurable training parameters
  - Comprehensive docstrings and inline comments

### Testing
- `tests/`: Test suite with pytest
  - `tests/__init__.py`: Empty init file
  - `tests/test_main.py`: 27 unit tests covering activation, prediction, logic gates, training, and determinism

### Examples
- `example/`: Demo scripts
  - `example/demo.py`: Example usage demonstrating training and prediction

### Configuration
- `pyproject.toml`: Project metadata, dependencies, build system, and tool configuration (ruff, etc.)
- `.pre-commit-config.yaml`: Pre-commit hooks for automated code quality checks
- `.gitignore`: Standard Python gitignore patterns

### Documentation
- `README.md`: Comprehensive documentation with usage, examples, and references
- `CODE_REVIEW.md`: Code review notes (if present)

### Generated/Cache Files
- `training.log`: Training output log (generated at runtime)
- `.coverage`: Coverage data from pytest-cov
- `.mypy_cache/`: Mypy type checking cache
- `.pytest_cache/`: Pytest cache
- `.ruff_cache/`: Ruff linting cache
- `__pycache__/`: Python bytecode cache
- `venv/`: Virtual environment (not tracked in git)
- `perceptron.egg-info/`: Package metadata (generated by setuptools)

## Code Organization Patterns

### main.py Structure
1. License header and module docstring
2. Imports (standard library only)
3. Constants (LOGIC_GATES dictionary)
4. Core functions (step, predict, visualize_decision_boundary)
5. Training and testing functions
6. CLI argument parsing and main() entry point

### Naming Conventions
- Functions: lowercase with underscores (e.g., `step`, `predict`, `train`)
- Variables: lowercase with underscores (e.g., `learning_rate`, `max_epochs`)
- Constants: UPPERCASE (e.g., `LOGIC_GATES`)
- Type hints used throughout for clarity

### Testing Patterns
- Test classes group related tests (e.g., `TestActivationFunction`, `TestTrainingFunction`)
- Test methods prefixed with `test_`
- Descriptive test names indicating what is being tested
- Tests verify both expected convergence (AND, OR, NAND, NOR) and non-convergence (XOR)
